Exploding Ninja

This project was made in a group at the conclusion of the MEAN stack for Coding Dojo. The goal of this project is to adapt the popular board game “Exploding Kittens” into a playable web application. We want players to be able to invite friends and play with them. We chose to build this application using socket.io to allow for instant interaction between players. The front end is Javascript and the backend is Node and Socket.io. Every interaction with the game board is sent back to a server and emitted out to every other player.

The challange with this project was that the socket listeners and emitters were all on the same file. In hindsight we should have modularized this out. This made using this project with git very difficult since all members of the group were editing the same two files. We learned how important bieng able to modularize your code out when working in groups.

Building Plugins with Python Scripting

In a previous job I was tasked with investigating the roads around specific points at a detailed scale. The workflow consisted of importing Tiger 2015 road shapefiles into QGIS. Followed by zooming to the specified coordinates to inquire about the nearby roads. Loading one at a time was not practical, given that there were multiple coordinates to be searched. I built this plugin using templates online to help streamline the process.

Check out the live example.
See the Code.

Previous Workflow Workflow with the addandzoom Plugin
  • Load in ESRI Street basemap tiles with python for spatial reference
  • Load in county polygons shapefile (TIGER2015_county_index)
  • Symbolize with predefined template
  • Flip coordinate pair from Lat/Long to Long/Lat, and paste in the coordinates to zoom to the desired location
  • Use the identify tool on the county index polygon layer to find the code of the county which the point is located in
  • Search for the code among the names in the tiger roads directory
  • Add in the coresponding Tiger Roads shapefile that coresponds to the county that the point is in
  • Label and style roads with predefined template for easier readability


This process is not feasible given that the points can fall in random counties that span across the United States. Having all the roads already imported world slow down the map. More so, I might not even use half of the imported roads.
  • Start Python Console, Ctrl + Alt + P
  • Click addandzoom plugin button
  • Paste in coordinates, click OK


How it works ?

  1. Adds county index layer to QGIS and symbolizes
  2. Creates a point from the coordinates that were given to it
  3. Selects by location, to find which county contains the created point
  4. Takes the county code from the attributes of the selected county
  5. Adds in the appropriate Tiger road shapefile using the identified county code from the previous step
  6. It names the layer in the table of contents, labels the Tiger roads by road name, and symbolizses the roads.
  7. Lastly it adds a basemap for spatial reference

The plugin streamlines several steps, it adds in the data and zooms to the location with the push of a button. When the tool is given a new pair of coordinates, it removes the tiger roads from the previous use to reduce clutter. It also takes the coordinate data in its natural order Lat/Long, rather than having to flip the order so that QGIS can zoom to it. This tool allowed me a more effecient workflow.
